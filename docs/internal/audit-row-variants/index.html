Rouse Context Audit Row Variants (#343)

Generated by AuditRowVariantsScreenshotTest. Seven candidate client-label placement options for ToolCallRow (see #343). Each variant renders three sample client labels — short (Claude), long (Claude Desktop), and unknown (Unknown (#1)) — in both light and dark theme. Production ToolCallRow is not modified.

1. Third row, full-width

Client shown as a labelSmall line beneath the existing duration+args row. Keeps the top row untouched; every row gets its own label.

2. Second-row prefix

Client label prefixes the duration+args row: "Claude · 142ms · {days: 7}". Cheapest change; no new rows.

3. Right-aligned pill

Client rendered as a pill/chip between the tool name and the time. Top row gets busier; scanability depends on label length.

4. Leading initial chip

Circular initial chip (first letter, deterministic color) leads the row. Strongest visual anchor; needs a color per client.

5. Session-grouped header

Rows grouped under a session header that shows the client label once. Cleanest rows, but loses per-row context at a glance.

6. Row label + session header

Variant 1 + Variant 5: session header AND per-row client label. Redundant on purpose for scan-by-row scenarios.

7. Right-aligned pill with time below

Client pill sits alone at the top-right of row 1; the timestamp right-aligns on row 2 directly beneath it. Keeps the pill prominent without crowding the time next to it.
